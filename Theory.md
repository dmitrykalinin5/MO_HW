## ДЗ1.1. Кубическая аппроксимация

### Задача

Вариант 9:

\[
f(x)=\frac{x^3}{3}-5x+x\ln x, \quad [a,b]=[1.5,2].
\]

Производные:

\[
f'(x)=x^2+\ln x-4,
\]

\[
f''(x)=2x+\frac{1}{x}.
\]

На отрезке `[1.5, 2]` вторая производная положительна, значит функция выпуклая и минимум единственный.

Проверка концов:

- `f'(1.5) = -1.344535 < 0`
- `f'(2) = 0.693147 > 0`

Значит корень `f'(x)=0`, то есть точка минимума, находится внутри отрезка.

### Формула кубической аппроксимации

На текущем отрезке `[u, v]` строится кубический многочлен Эрмита по значениям:

- `f(u)`
- `f(v)`
- `f'(u)`
- `f'(v)`

Используемые формулы:

\[
w=f'(v)+f'(u)-3\frac{f(v)-f(u)}{v-u}
\]

\[
z=\sqrt{w^2-f'(u)f'(v)}
\]

\[
x^*=v-(v-u)\frac{f'(v)+z-w}{f'(v)-f'(u)+2z}
\]

Если `f'(x*) > 0`, минимум левее `x*`, поэтому обновляем `v = x*`.
Если `f'(x*) < 0`, минимум правее `x*`, поэтому обновляем `u = x*`.

Критерий остановки:

\[
|f'(x^*)| < 10^{-4}.
\]

### Ответ

Метод сошелся за 2 итерации:

\[
x_{min}\approx 1.841097,
\]

\[
f(x_{min})\approx -6.001533.
\]

Контроль точности:

\[
|f'(x^*)|\approx 4.58\cdot 10^{-8}<10^{-4}.
\]

### Возможные вопросы по ДЗ №1

**Почему минимум единственный?**
Потому что `f''(x)=2x+1/x > 0` на всем отрезке. Функция строго выпукла.

**Зачем нужны производные на концах отрезка?**
Метод кубической аппроксимации строит кубический многочлен Эрмита: ему нужны значения функции и первые производные в двух точках.

**Почему можно искать минимум через `f'(x)=0`?**
Для внутренней точки минимума дифференцируемой функции необходимое условие экстремума: `f'(x)=0`. Здесь минимум действительно внутренний, потому что производная меняет знак с минуса на плюс.

**Почему после первой итерации левая граница стала `1.8410219`?**
Потому что в новой точке производная еще отрицательная. Значит функция продолжает убывать, и минимум расположен правее этой точки.

**Что изображено на графике?**
Синяя линия — исходная функция. Красная пунктирная линия — кубический интерполяционный многочлен текущей итерации. Оранжевая звезда — найденная стационарная точка кубики.

## ДЗ №2.1. Методы оптимизации из ЛР4

### Задача

Вариант 9:

\[
F(x_1,x_2)=3x_1^2+2x_1x_2+3x_2^2-4x_1-12x_2+12.
\]

Начальная точка:

\[
M_0=(-3,-3).
\]

Градиент:

\[
\nabla F =
\begin{pmatrix}
6x_1+2x_2-4\\
2x_1+6x_2-12
\end{pmatrix}.
\]

Гессиан:

\[
H=\begin{pmatrix}6&2\\2&6\end{pmatrix}.
\]

Собственные значения Гессиана: `4` и `8`, оба положительные. Значит функция строго выпуклая.

Аналитический минимум находится из `grad F = 0`:

\[
(x_1^*,x_2^*)=(0,2), \quad F(0,2)=0.
\]

### Покоординатный спуск

Идея: по очереди минимизируем функцию по одной координате, фиксируя остальные.

Для этой функции точные формулы координатного минимума:

\[
x_1=\frac{2-x_2}{3},
\]

\[
x_2=\frac{6-x_1}{3}.
\]

Результат:

- шагов: `14`
- финальная точка: `(0.000003, 1.999999)`
- значение функции: `2.62e-11`
- норма градиента: `1.67e-05`

### Градиентный спуск

Формула:

\[
x^{k+1}=x^k-\alpha_k\nabla F(x^k).
\]

В реализации начальный шаг `alpha = 0.1`. Если значение функции не убывает, шаг делится пополам.

Результат:

- шагов: `22`
- финальная точка: `(0.000013, 1.999987)`
- значение функции: `6.93e-10`
- норма градиента: `7.45e-05`

### Наискорейший спуск

Направление — антиградиент, как и в градиентном спуске. Отличие — шаг выбирается оптимально вдоль этого направления.

Для квадратичной функции:

\[
\alpha_k=\frac{g_k^Tg_k}{g_k^THg_k}, \quad g_k=\nabla F(x^k).
\]

Результат:

- шагов: `6`
- финальная точка: `(-0.000001, 1.999998)`
- значение функции: `2.38e-11`
- норма градиента: `1.94e-05`

### Возможные вопросы по ДЗ №2.1

**Ты использовал `Contours and solutions.ipynb`?**
Да. Из notebook взята базовая схема построения: создается сетка `meshgrid`, на ней считается функция, через `contour` строятся линии уровня, а поверх них рисуется ломаная последовательности точек. В моей программе это адаптировано под вариант 9 и изменено по требованию ДЗ2.1: уровни берутся из значений функции в вершинах траектории, а не задаются просто как `levels=50`.

**Почему все методы сошлись в одну точку?**
Потому что функция строго выпуклая, а значит у нее единственный глобальный минимум.

**Почему линии уровня — эллипсы?**
Потому что квадратичная форма положительно определена. Уравнение `F(x1,x2)=const` задает эллипсы вокруг минимума.

**Почему траектория покоординатного спуска ломаная с прямыми углами?**
Потому что метод двигается по одной координатной оси за раз: сначала меняет `x1`, затем `x2`.

**Чем градиентный спуск отличается от наискорейшего?**
В градиентном спуске шаг задается заранее или подбирается простым правилом. В наискорейшем спуске шаг каждый раз выбирается как минимум функции вдоль направления антиградиента.

**Почему для наискорейшего спуска шаг считается такой формулой?**
Для квадратичной функции значение вдоль прямой `x - alpha*g` является одномерной квадратичной функцией. Ее минимум находится аналитически, отсюда формула `alpha = (g^T g)/(g^T H g)`.

## ДЗ №2.2. Метод Ньютона

### Задача

Вариант 9:

\[
z(x,y)=x^3-6x^2+9x+y^3-11y^2+39y-49.
\]

Начальная точка:

\[
(2.5,4.5).
\]

Градиент:

\[
\nabla z =
\begin{pmatrix}
3x^2-12x+9\\
3y^2-22y+39
\end{pmatrix}.
\]

Гессиан:

\[
H =
\begin{pmatrix}
6x-12 & 0\\
0 & 6y-22
\end{pmatrix}.
\]

### Стационарные точки

Из `grad z = 0`:

\[
3x^2-12x+9=0 \Rightarrow x\in\{1,3\}
\]

\[
3y^2-22y+39=0 \Rightarrow y\in\{3,13/3\}
\]

Стационарные точки:

| Точка | Значение | Тип |
|---|---:|---|
| `(1, 3)` | `0` | локальный максимум |
| `(1, 13/3)` | `-1.185185` | седловая точка |
| `(3, 3)` | `-4` | седловая точка |
| `(3, 13/3)` | `-5.185185` | локальный минимум |

Классификация выполнена по знакам собственных значений матрицы Гессе.

### Метод Ньютона

Формула:

\[
X_{k+1}=X_k-H(X_k)^{-1}\nabla z(X_k).
\]

Критерий остановки:

\[
\|\nabla z(X_k)\|<10^{-4}.
\]

Итерации привели к точке:

\[
(x^*,y^*)\approx(3,4.333333)=\left(3,\frac{13}{3}\right).
\]

Значение:

\[
z(x^*,y^*)\approx -5.185185=-\frac{140}{27}.
\]

Норма градиента на последней итерации:

\[
2.79\cdot10^{-7}<10^{-4}.
\]

### Возможные вопросы по ДЗ №2.2

**Почему точка `(3, 13/3)` — минимум?**
В этой точке Гессиан равен `diag(6, 4)`, оба собственных значения положительные. Значит Гессиан положительно определен, а точка — локальный минимум.

**Почему `(1, 3)` — максимум?**
Гессиан `diag(-6, -4)` отрицательно определен, значит это локальный максимум.

**Почему две остальные точки седловые?**
У Гессиана собственные значения разных знаков. Значит функция в одних направлениях возрастает, а в других убывает.

**Почему Ньютон сошелся именно к минимуму, а не к другой стационарной точке?**
Начальная точка `(2.5, 4.5)` находится в области притяжения стационарной точки `(3, 13/3)`. Итерации Ньютона быстро уточняют корень системы `grad z = 0`.

**Метод Ньютона всегда ищет минимум?**
Нет. Формально он ищет стационарную точку, то есть корень градиента. Поэтому после нахождения точки нужно классифицировать ее через Гессиан.

## ДЗ №2.3. Объектная модель

### Сущности

В `dz2_3_object_model.py` введены классы:

- `Objective2D` — абстрактная целевая функция с методами `value`, `gradient`, `hessian`.
- `Variant9Quadratic` — конкретная квадратичная функция варианта 9.
- `Optimizer2D` — абстрактный оптимизатор.
- `CoordinateDescent` — покоординатный спуск.
- `GradientDescent` — градиентный спуск.
- `SteepestDescent` — наискорейший спуск.
- `OptimizationResult` — результат оптимизации: путь, значения функции, нормы градиента.

### Отношения

- Наследование: конкретные оптимизаторы наследуются от `Optimizer2D`.
- Композиция: оптимизатор содержит объект целевой функции.
- Агрегация: результат содержит список точек траектории.
- Использование: методы оптимизации вызывают `value`, `gradient`, `hessian` у целевой функции.

### Возможные вопросы по объектной модели

**Зачем нужен абстрактный класс `Objective2D`?**
Чтобы оптимизаторы не зависели от конкретной функции. Им достаточно знать, как получить значение, градиент и Гессиан.

**Почему результат вынесен в отдельный класс?**
Так удобнее передавать траекторию в визуализатор и отчет: неважно, какой метод породил путь.

**Где тут наследование?**
`CoordinateDescent`, `GradientDescent`, `SteepestDescent` наследуются от `Optimizer2D`. `Variant9Quadratic` наследуется от `Objective2D`.

**Где композиция?**
Каждый оптимизатор хранит ссылку на объект целевой функции, с которой работает.
